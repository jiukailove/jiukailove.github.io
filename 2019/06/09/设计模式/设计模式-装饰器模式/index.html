设计模式-装饰器模式

发表于2019-06-09|设计模式
|阅读量:

一、定义

  • 结构型模式
  • 装饰器模式允许向现有的对象添加新的功能，同时又不改变其结构
  • 创建一个装饰类，用来包装原有的类，并在保持类方法签名完整性的前提下，提供了额外的功能

二、角色

Component（抽象构件）：它是具体构件和抽象装饰类的共同父类，声明了在具体构件中实现的业务方法，它的引入可以使客户端以一致的方式处理未被装饰的对象以及装饰之后的对象，实现客户端的透明操作。

ConcreteComponent（具体构件）：它是抽象构件类的子类，用于定义具体的构件对象，实现了在抽象构件中声明的方法，装饰器可以给它增加额外的职责（方法）。

Decorator（抽象装饰类）：它也是抽象构件类的子类，用于给具体构件增加职责，但是具体职责在其子类中实现。它维护一个指向抽象构件对象的引用，通过该引用可以调用装饰之前构件对象的方法，并通过其子类扩展该方法，以达到装饰的目的。

ConcreteDecorator（具体装饰类）：它是抽象装饰类的子类，负责向构件添加新的职责。每一个具体装饰类都定义了一些新的行为，它可以调用在抽象装饰类中定义的方法，并可以增加新的方法用以扩充对象的行为。

具体构件类和装饰类都实现了相同的抽象构件接口，因此装饰模式以对客户透明的方式动态地给一个对象附加上更多的责任。

三、实现

  1. 创建接口
public interface Shape {
    void draw();
}
  1. 创建接口实现类
public class Rectangle implements Shape {
    @Override
    public void draw() {
        System.out.println("I'm Rectangle");
    }
}
  1. 创建接口的抽象装饰类
public class ShapeDecorator implements Shape {

    private Shape decoratedShape;

    public ShapeDecorator(Shape decoratedShape) {
        this.decoratedShape = decoratedShape;
    }

    @Override
    public void draw() {
        decoratedShape.draw();
    }
}
  1. 创建具体的抽象装饰类
//第一个装饰类
public class RedShapeDecorator extends ShapeDecorator {
    public RedShapeDecorator(Shape decoratedShape) {
        super(decoratedShape);
    }

    @Override
    public void draw() {
        super.draw();
        System.out.println("I'am redDecorator");
    }
}

//第二个装饰类
public class BlueShapeDecorator extends ShapeDecorator {
    public BlueShapeDecorator(Shape decoratedShape) {
        super(decoratedShape);
    }

    @Override
    public void draw() {
        super.draw();
        System.out.println("I'm blueDecorator");
    }
}
  1. 用装饰类来装饰对象
public class Client {
    public static void main(String[] args) {
        Shape rectangle = new Rectangle();
        //一层装饰
        ShapeDecorator redShapeDecorator = new RedShapeDecorator(rectangle);
        redShapeDecorator.draw();
        System.out.println("----------------分割线--------------");
        //两层装饰
        ShapeDecorator blueDecorator = new BlueShapeDecorator(redShapeDecorator);
        blueDecorator.draw();
    }
}

执行结果：
I'm Rectangle
I'am redDecorator
----------------分割线--------------
I'm Rectangle
I'am redDecorator
I'am blueDecorator

  • 装饰类与被装饰类实现统一接口
  • 装饰类持有被装饰类的引用

装饰器模式与静态代理类似，但静态代理不一定要满足is-a的关系，静态代理关注的是对代理对象的控制，而装饰器模式关注的是自身功能的扩展。

拿健身的例子来说：

装饰器模式相当于自己吃蛋白粉，健身，自己身体变的强壮了，自己能保护自己，还是一个人独闯天下；

代理模式详单于自己请了个保镖保护自己，自己的体格还是那个体格，但是出去闯荡社会不可能有人欺负咱了。
