设计模式-代理模式

发表于2019-06-08|设计模式
|阅读量:

定义

代理模式(Proxy Pattern)指为其他对象提供一种代理，以控制对这个对象的访问，属于结构性设计模式。

在某些情况下，一个对象不适合或者不能直接饮用另一个对象，而代理对象可以在客户端与目标对象之间起到中介的作用。

静态代理

public class Client {

    public static void main(String[] args) {
        UserServiceProxy proxy = new UserServiceProxy(new UserServiceImpl());
        proxy.update();
    }

}

interface UserService {
    void update();
}

class UserServiceImpl implements UserService {

    @Override
    public void update() {
        System.out.println("do update");
    }
}

class UserServiceProxy implements UserService {

    private UserService userService;

    public UserServiceProxy(UserService userService) {
        this.userService = userService;
    }

    @Override
    public void update() {
        before();
        userService.update();
        after();
    }

    private void before() {
        System.out.println("before update");
    }

    private void after() {
        System.out.println("after update");
    }
}

虽然静态代理实现简单，且不侵入原代码，但是当场景稍微复杂一些的时候，静态代理的缺点也会显露出来，主要体现在：

  • 当需要代理多个类的时候，需要实现多个接口或者创建多个代理类
  • 当接口需要修改的时候，目标对象和代理类都要同时修改，不易维护

动态代理

JDK动态代理

public class Client {
    public static void main(String[] args) {
        //被代理对象
        UserServiceImpl userService = new UserServiceImpl();
        ClassLoader classLoader = userService.getClass().getClassLoader();
        Class<?>[] interfaces = userService.getClass().getInterfaces();
        LogHandler handler = new LogHandler(userService);

        UserService proxy = (UserService) Proxy.newProxyInstance(classLoader, interfaces, handler);
        proxy.update();
    }
}

public class LogHandler implements InvocationHandler {

    //被代理对象
    private Object target;

    public LogHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        before();
        Object result = method.invoke(target, args);
        after();
        return result;
    }

    private void before() {
        System.out.println("before update");
    }

    private void after() {
        System.out.println("after update");
    }

}

public interface UserService {
    void update();
}

public class UserServiceImpl implements UserService {
    @Override
    public void update() {
        System.out.println("do update");
    }
}

也可以在直接handler中定义方法，返回代理对象

public class Client {
    public static void main(String[] args) {
        UserServiceProxy proxy = new UserServiceProxy();
        UserService userService = proxy.getInstance(new UserServiceImpl());
        userService.update();
    }
}

public class UserServiceProxy implements InvocationHandler {

    private UserService target;

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        before();
        Object result = method.invoke(target, args);
        after();
        return result;
    }

    private void before() {
        System.out.println("before update");
    }

    private void after() {
        System.out.println("after update");
    }

    public UserService getInstance(UserService userService) {
        target = userService;
        Class<? extends UserService> clazz = target.getClass();
        return (UserService) Proxy.newProxyInstance(clazz.getClassLoader(), clazz.getInterfaces(), this);
    }

}

CGLIB动态代理

与JDK动态代理极为相似，甚至更加简单。

public class CglibClient {
    public static void main(String[] args) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(UserDao.class);
        enhancer.setCallback(new LogInterceptor());
        UserDao proxy = (UserDao) enhancer.create();
        proxy.update();
    }
}

public class UserDao {
    public void update() {
        System.out.println("do update");
    }
}

public class LogInterceptor implements MethodInterceptor {

    @Override
    public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {
        before();
        Object result = methodProxy.invokeSuper(o, objects);
        after();
        return result;
    }

    private void before() {
        System.out.println("before update");
    }

    private void after() {
        System.out.println("after update");
    }
}

两种动态代理优缺点

JDK动态代理：

  • 基于Java反射机制实现

  • 无需额外依赖类库

  • 可平滑进行JDK版本升级

  • 需实现接口

CGLIB动态代理：

  • 基于ASM机制实现，通过生成业务类的子类作为代理类
  • 无需实现接口，达到代理类无侵入
  • 高性能
  • JDK升级时，字节码类库需要进行更新以保证在新版JDK上能够使用
